# Проєкт 06: Спостережуваність runtime та налагодження (фінальний проєкт)

Фінальний проєкт: побудова та бенчмаркінг повного harness, а потім запуск циклів очищення для верифікації якості та підтримуваності.

## Довідник по директоріях

| Директорія | Значення |
|------|------|
| `starter/` | **Відправна точка**: повний продуктовий код, але harness навмисно ослаблений (лише базовий AGENTS.md, без feature_list.json, передавання сесій або чеклиста чистого стану). |
| `solution/` | **Еталонна реалізація**: максимальний harness з усіма файлами артефактів, високими оцінками якісних документів, скриптами бенчмаркінгу та сканерами очищення. |

## Як використовувати

```sh
cd starter
npm install
# Запустіть застосунок і вручну зафіксуйте поведінку зі слабким harness.
# Стартер навмисно не містить benchmark.sh та cleanup-scanner.sh.

cd ../solution
npm install
# Запустіть той самий бенчмарк із повним harness
# Виконайте цикли очищення
# Порівняйте зміни оцінок у quality-document.md

# Запустіть тести бенчмаркінгу
./scripts/benchmark.sh

# Запустіть сканування очищення
./scripts/cleanup-scanner.sh
```

## Точний контракт завдання

Проєкт 06 — це фінальне порівняння між повним продуктом із слабким покриттям harness
та тим самим продуктом, зміцненим повним набором артефактів harness. На відміну від попередніх
проєктів, стартер вже містить більшість функціональності продукту. Прогалина — це
операційна система навколо коду.

| Область | Стан стартера | Докази рішення |
|------|------|------|
| Поведінка продукту | Імпорт, індексування, QA, історія, зворотний зв'язок, скидання здебільшого є | Ті самі функції плюс більш строга верифікація та докази збереження стану |
| Файли harness | Базовий `AGENTS.md`, немає `feature_list.json`, `session-handoff.md`, чеклиста чистого стану | `AGENTS.md`, `CLAUDE.md`, `feature_list.json`, `init.sh`, `session-handoff.md`, `clean-state-checklist.md` |
| Відстеження якості | Лише початковий `quality-document.md` | `quality-document.md` з вищою оцінкою, `evaluator-rubric.md` |
| Бенчмаркінг | Немає скриптів бенчмарку або очищення | `scripts/benchmark.sh`, `scripts/cleanup-scanner.sh`, `scripts/check-architecture.sh` |
| Документи надійності | Мінімальна документація | `docs/ARCHITECTURE.md`, `docs/PRODUCT.md`, `docs/RELIABILITY.md` |

Не очікуйте, що стартер міститиме команди бенчмаркінгу, показані для
рішення. Для запуску зі слабким harness фіксуйте вручну базові спостереження; для
запуску рішення використовуйте зафіксовані скрипти бенчмаркінгу та очищення.

## Охоплені функції

- Імпорт документів
- Побудова або оновлення індексу
- Відповіді на запитання з цитатами
- Зворотний зв'язок від runtime
- Читабельний стан репозиторію, з якого можна перезапустити роботу
