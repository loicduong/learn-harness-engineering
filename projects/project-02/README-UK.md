# Проєкт 02: Робочий простір, зрозумілий для агента

Продемонструйте, як читабельність репозиторію та явні артефакти безперервності зменшують втрату контексту під час розробки в кількох сесіях.

## Структура каталогів

| Каталог | Призначення |
|------|------|
| `starter/` | **Відправна точка**: базується на рішенні P1, де імпорт документів, детальний перегляд і збереження ще потрібно реалізувати. Harness слабкий: AGENTS.md мінімальний, а передачі сесії немає. |
| `solution/` | **Еталонна реалізація**: усі нові функції реалізовано, з повною документацією робочого простору (ARCHITECTURE.md, PRODUCT.md, session-handoff.md). |

## Як використовувати

```sh
# Для виконання потрібно щонайменше 2 сесії агента
cd starter
npm install
# Сесія A: реалізуйте імпорт документів і детальний перегляд
# Сесія B: реалізуйте збереження (спостерігайте, чи агент швидко відновлює контекст)

cd ../solution
npm install
# Повторіть запуск із повним harness і порівняйте швидкість відновлення сесії
```

## Точний контракт завдання

Почніть із `starter/`. Стартова точка вже містить оболонку Проєкту 01 і
мінімальний harness. Завдання — додати зріз продукту Проєкту 02 і
документацію робочого простору, яка дозволяє другій сесії швидко відновитися.

| Функція / артефакт | Стан у starter | Докази в solution |
|------|------|------|
| Імпорт документів | Потік імпорту є неповним у renderer, preload, IPC та `DocumentService` | `src/renderer/components/ImportPanel.tsx`, `src/main/ipc-handlers.ts`, `src/services/document-service.ts` |
| Деталі документа | Панель деталей не завантажує та не відображає повний вміст файлу через IPC | `src/renderer/components/DocumentDetail.tsx`, `IPC_CHANNELS.GET_DOCUMENT_CONTENT` |
| Базове збереження | Імпортовані документи не відновлюються повністю після перезапуску | `src/services/persistence-service.ts`, обробка `documents-meta.json` |
| Стан, зрозумілий для агента | У starter немає фінального файлу передачі | `session-handoff.md`, розширений `docs/ARCHITECTURE.md`, розширений `docs/PRODUCT.md` |

Виконуйте це як вправу в двох сесіях. Зупиніть Сесію A до завершення всіх трьох
функцій продукту, потім розпочніть Сесію B лише зі стану репозиторію. Головне
порівняння — наскільки швидше Сесія B може відновити роботу, коли `session-handoff.md`
і документація існують.

## Охоплені функції

- Потік імпорту документів (вибір файлу та передача через IPC)
- Детальний перегляд документа (відображення метаданих і вмісту)
- Базове збереження (імпортовані документи залишаються після перезапуску)
