# Проєкт 03: Безперервність кількох сесій і контроль обсягу

Оцініть, чи покращують журнали прогресу, передача сесії, явний контроль обсягу та
верифікаційні вентилі точність виконання після перезапусків.

## Структура каталогів

| Каталог | Призначення |
|------|------|
| `starter/` | **Відправна точка**: базується на рішенні P2, де поділ документів на частини, витягування метаданих, статус індексу та обґрунтований QA ще потрібно реалізувати. Містить стартовий `feature_list.json`, але бракує повного harness перезапуску (`init.sh`, `session-handoff.md`, `claude-progress.md`, контрольного списку чистого стану). |
| `solution/` | **Еталонна реалізація**: усі функції реалізовано. AGENTS.md включає стратегію «одна функція за раз», а репозиторій доповнено артефактами перезапуску/безперервності (`init.sh`, `session-handoff.md`, `claude-progress.md`, `clean-state-checklist.md`). |

## Як використовувати

```sh
cd starter
npm install
# Запустіть щонайменше дві сесії агента. Зупиніть посередині задачі, потім відновіть зі стану репозиторію.
# Спостерігайте, чи агент зберігає обсяг і оновлює feature_list.json.

cd ../solution
npm install
# Перегляньте завершені артефакти безперервності та докази функцій.
```

## Точний контракт завдання

Стартова точка Проєкту 03 — не порожній застосунок. Це Проєкт 02 плюс незавершена
робота з індексування та обґрунтованого QA. Виконуйте ці пункти по одному за раз:

| Функція / артефакт | Стан у starter | Докази в solution |
|------|------|------|
| Поділ документів на частини | `IndexingService` потребує створення частин з урахуванням абзаців | `src/services/indexing-service.ts`, елемент `document-chunking` у feature_list.json |
| Витягування метаданих | Метадані документа є неповними для роботи з індексуванням | `src/services/document-service.ts`, `DocumentDetail.tsx`, елемент `metadata-extraction` |
| UI статусу індексування | Рядок статусу потребує кількості проіндексованих / загальної кількості частин / кольорів статусу | `src/renderer/components/StatusBar.tsx`, `App.tsx`, елемент `indexing-status-ui` |
| Обґрунтований QA | QA має цитувати отримані частини з уривками та рівнем впевненості | `src/services/qa-service.ts`, `QuestionPanel.tsx`, елемент `grounded-qa` |
| Harness безперервності | У starter бракує фінальних артефактів перезапуску/передачі | `init.sh`, `session-handoff.md`, `claude-progress.md`, `clean-state-checklist.md` |

Не трактуйте це як «додай будь-яку функцію для кількох сесій». Запланований зріз
продукту — індексування плюс QA із посиланнями на джерела, а запланований зріз
harness — відстеження прогресу з можливістю перезапуску.

## Охоплені функції

- Поділ документів на частини (з урахуванням абзаців, близько 500 символів)
- Витягування метаданих (кількість слів, рядків, абзаців)
- Статус індексу, відображений в UI
- Базовий потік QA із посиланнями на джерела
