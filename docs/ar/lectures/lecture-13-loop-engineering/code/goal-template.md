## النطاق

<!-- كن صريحًا حول ما لا يجب لمسه -->

### مسموح به

- جميع الملفات ضمن `src/xx/`
- ملفات الاختبار ضمن `tests/`
- ملفات تعريف الأنواع ذات الصلة

### ممنوع لمسه

- ملف الإدخال `src/main.ts`
- ترحيلات مخطط قاعدة البيانات
- إصدارات التبعيات في `package.json` (إلا إذا كانت مطلوبة بشكل صريح)
- ملفات تكوين CI/CD

## طريقة التحقق

<!-- كيف تتحقق بعد كل خطوة -->

1. بعد التنفيذ، شغّل هذه الأوامر بالترتيب:
   1. `npx tsc --noEmit` — فحص الأنواع
   2. `npm run lint` — أسلوب الكود
   3. `npm test` — اختبارات الوحدة
   4. `npx vitest --coverage` — فحص التغطية

2. أصلح أي إخفاقات قبل الانتقال إلى الخطوة التالية.
3. عندما يمر كل شيء، شغّل مجموعة اختبار الشاملة الكاملة (إذا كانت متوفرة).

## شروط التوقف

- جميع معايير القبول تمر ✅
- الوصول إلى الحد الأقصى من الدورات: 20
- لا تقدم لمدة 3 جولات متتالية (يظل نفس الخطأ يظهر)
- مشكلة حاجزة لا يمكن حلها بشكل مستقل (مثل تبعية مفقودة، مشكلة بيئة)

## كيفية العمل

1. اقرأ `AGENTS.md` و `feature_list.json` أولاً لفهم بنية المشروع والميزات الحالية.
2. اكتب نهج التصميم قبل لمس الكود.
3. تحقق بعد انتهاء كل مهمة فرعية.
4. إذا تعثرت لأكثر من جولتين، غيّر النهج أو بسّط.
5. حدّث التقدم في نهاية كل جولة.
