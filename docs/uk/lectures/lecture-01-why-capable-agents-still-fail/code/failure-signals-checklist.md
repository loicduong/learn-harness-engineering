# Чекліст сигналів збою

Використовуйте цей чекліст під час аналізу запуску зі слабким harness.

- Чи запитував агент, або невірно здогадувався, як запустити застосунок?
- Чи створював він директорії або абстракції, що не відповідають задуманому продукту?
- Чи зупинявся він після створення видимої UI-оболонки без повноцінного робочого процесу?
- Чи залишав він нотатки або артефакти, що допомогли б наступному запуску продовжити роботу?
- Чи зможе нова сесія зрозуміти, що відбулося, менш ніж за п'ять хвилин?
