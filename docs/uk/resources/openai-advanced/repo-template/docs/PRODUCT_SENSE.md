# PRODUCT_SENSE.md

Цей файл фіксує тривале продуктове судження, яке агенти не можуть надійно
вивести лише з коду.

## Ядро продукту

- Основний користувач: `[замінити]`
- Завдання, що виконується: `[замінити]`
- Основне розчарування, яке потрібно усунути: `[замінити]`
- Планка якості для прийнятності: `[замінити]`

## Правила продукту

- Надавайте перевагу надійності, видимій користувачеві, над кількістю функцій.
- Розглядайте неоднозначну поведінку як прогалину в специфікації, а не як дозвіл вгадувати.
- Якщо реалізація змінює те, що бачать або чому довіряють користувачі, оновіть відповідну специфікацію.
- Використовуйте специфікації продукту для конкретних потоків, а цей файл — для наскрізних
  продуктових пріоритетів.

## Заборонені патерни

- Приховані деструктивні дії
- Тихі збої без зворотного зв'язку для користувача
- Незрозуміле джерело істини для видимого стану
- Функції, які неможливо пояснити одним реченням
