# PLANS.md

Цей файл визначає, як плани виконання створюються, оновлюються, завершуються та
архівуються.

## Коли потрібен план

Створюйте план виконання, якщо робота:

- охоплює більше однієї сесії
- змінює більше однієї підсистеми
- має нетривіальну верифікацію або ризики розгортання
- залежить від відкритих рішень, які мають бути зафіксовані

## Розташування планів

- `docs/exec-plans/active/`: плани, що наразі керують роботою
- `docs/exec-plans/completed/`: завершені плани, збережені для майбутнього контексту агентів
- `docs/exec-plans/tech-debt-tracker.md`: відкладена робота та подальші дії

## Мінімальні розділи плану

- мета
- обсяг та поза обсягом
- шлях верифікації
- ризики та блокери
- журнал прогресу
- відкриті рішення

## Операційні правила

- Один активний план повинен мати один чітко визначений поточний крок.
- Оновлюйте план по ходу роботи; не ставтеся до нього як до статичного тексту.
- Якщо рішення змінює напрям реалізації, зафіксуйте це в плані.
- Переміщуйте завершені плани до `completed/`, щоб агенти могли знаходити попередній контекст.
