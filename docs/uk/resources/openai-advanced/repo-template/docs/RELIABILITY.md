# RELIABILITY.md

Цей файл визначає, як система доводить, що вона здорова та може перезапуститися.

## Стандартні шляхи

- Завантаження: `[command]`
- Верифікація: `[command]`
- Запуск застосунку або сервісу: `[command]`
- Налагодження або інспекція runtime: `[command]`

## Обов'язкові сигнали Runtime

- структуровані журнали для запуску та критичних потоків
- перевірки стану здоров'я для ключових сервісів
- дані трасування або таймінгу для повільних шляхів, коли доступні
- стани помилок, видимі користувачеві, для відновлюваних збоїв

## Золоті шляхи

- `[шлях 1]`
- `[шлях 2]`
- `[шлях 3]`

Кожен золотий шлях повинен мати повторюваний шлях верифікації та чіткі сигнали збою.

## Правила надійності

- Жодна функція не є завершеною, якщо після неї система не може чисто перезапуститися.
- Збої runtime мають діагностуватися за сигналами, доступними локально в репозиторії.
- Якщо з'являється повторюваний режим збою, додайте для нього бенчмарк або запобіжний захід.
- Очищення є частиною надійності, а не окремою проблемою.
