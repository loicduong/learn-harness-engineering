# FRONTEND.md

Цей файл визначає стабільні очікування frontend, щоб агенти не вигадували UI-патерни
непередбачувано.

## Принципи UI

- Оптимізуйте для ясності, а не новизни.
- Тримайте інтерфейсні потоки такими, що легко знайти та можна перезапустити.
- Надавайте перевагу невеликій кількості повторно використовуваних компонентів замість
  одноразових варіантів.
- Перевірки доступності є частиною звичайної верифікації, а не роботи з поліруванням.

## Запобіжні заходи

- Документуйте дизайн-систему або бібліотеку компонентів у `docs/references/`.
- Фіксуйте ключові стани, видимі користувачеві: порожній, завантаження, успіх, помилка, повтор.
- Тримайте тексти, поведінку клавіатури та візуальну ієрархію послідовними між потоками.
- Коли виправляєте UI-помилку, додайте або оновіть відповідний крок валідації.

## Очікування верифікації

- Збирайте докази для критичних шляхів користувача.
- Фіксуйте кроки валідації браузера або runtime у відповідному плані.
- Якщо візуальні регресії є частими, стандартизуйте перевірки знімків екрана або DOM.
