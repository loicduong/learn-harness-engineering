# DESIGN.md

Цей файл є точкою входу для проєктування. Тримайте його коротким і використовуйте для
маршрутизації до більш детальних файлів у `docs/design-docs/`.

## Призначення

Фіксуйте тривалі рішення щодо продукту та системного проєктування, які мають
пережити одну розмову, спринт або пам'ять рецензента.

## Читайте цей файл, коли

- вам потрібна поточна філософія проєктування
- ви збираєтеся ввести новий патерн
- вам потрібно знати, які рішення щодо проєктування є остаточними, а які ще відкриті

## Канонічні документи проєктування

- `docs/design-docs/index.md`: індекс прийнятих, запропонованих і застарілих документів
- `docs/design-docs/core-beliefs.md`: загальні для проєкту принципи, орієнтовані на агентів

## Правила проєктування

- Тримайте документи проєктування невеликими й актуальними.
- Надавайте перевагу одному документу на область рішення.
- Пов'язуйте документи проєктування з планами та специфікаціями, коли зміна залежить від них.
- Якщо правило проєктування стає операційно критичним, перетворіть його на автоматизовану
  перевірку або оновіть `ARCHITECTURE.md`.
