# OpenAI Advanced SOPs

Ці SOP перекладають операційні патерни зі статті в конкретні плейбуки
виконання, яким можна слідувати або які можна адаптувати.

## Включені SOP

- [`layered-domain-architecture.md`](./layered-domain-architecture.md):
  встановити явні шари та міжшарові межі
- [`encode-knowledge-into-repo.md`](./encode-knowledge-into-repo.md):
  перенести невидимі знання з чатів, документів і пам'яті у файли всередині репозиторію
- [`observability-feedback-loop.md`](./observability-feedback-loop.md):
  надати агентам логи, метрики, трейси та повторюваний цикл налагодження
- [`chrome-devtools-validation-loop.md`](./chrome-devtools-validation-loop.md):
  використовувати браузерну автоматизацію та знімки для валідації UI-поведінки до досягнення чистого стану

## Як ними користуватися

1. Виберіть SOP, що відповідає вашому поточному вузькому місцю.
2. Використовуйте контрольний список для налаштування відсутніх артефактів або інструментів.
3. Зафіксуйте отримані правила у скопійованих документах `repo-template/`.
4. Перетворюйте повторювані зауваження на рев'ю у перевірки, скрипти або захисні бар'єри.

Ці процедури не призначені для сліпого виконання. Вони покликані зробити harness
більш прозорим, керованим і відтворюваним.
