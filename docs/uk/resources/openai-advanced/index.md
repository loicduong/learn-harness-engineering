# OpenAI Advanced Pack

Ця тека містить більш структурований вигляд репозиторію, описаний у статті
OpenAI "Harness engineering: leveraging Codex in an agent-first world",
упакований у готові до копіювання стартові файли.

Використовуйте цей пакет, коли мінімального harness вже недостатньо і ваш репозиторій
потребує:

- короткого `AGENTS.md` у стилі маршрутизатора
- постійних документів-системи-обліку всередині репозиторію
- активних і завершених планів виконання
- явних файлів політики продукту, надійності, безпеки та фронтенду
- оцінки якості за продуктовим доменом і архітектурним шаром
- теок з довідковими матеріалами, зручними для моделей
- стандартних операційних процедур для архітектури, фіксації знань і верифікації runtime

## Як прийняти цей пакет

1. Почніть з мінімального пакету, якщо репозиторій ще невеликий.
2. Скопіюйте файли з [`repo-template/`](./repo-template/index.md) у власний
   репозиторій, коли потрібна міцніша структура.
3. Тримайте `AGENTS.md` коротким. Розглядайте його як маршрутизатор до глибших
   документів, а не як енциклопедію.
4. Оновлюйте документи якості, надійності та планів у процесі звичайної роботи,
   а не в окремий день прибирання.
5. Робіть згенеровані артефакти та зовнішні посилання явними, щоб агенти могли
   їх знаходити без опори на історію чатів.

## Бібліотека SOP

Тека [`sops/`](./sops/index.md) перетворює діаграми зі статті на
покрокові операційні процедури:

- налаштування багаторівневої доменної архітектури
- фіксація прихованих знань у репозиторії
- локальний стек спостережуваності та робочий процес зворотного зв'язку
- цикл валідації через Chrome DevTools для UI-роботи

## Принципи проектування

- Короткий вхідний пункт, глибші пов'язані документи
- Репозиторій як система обліку
- Механічні перевірки перемагають запам'ятовані правила
- Плани та історія якості живуть поруч із кодом
- Прибирання та спрощення є відповідальністю першого рівня

Цей пакет є навмисно структурованим, але його все одно слід адаптувати під ваш
проєкт, а не копіювати сліпо.
