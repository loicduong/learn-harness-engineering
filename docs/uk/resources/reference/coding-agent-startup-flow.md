# Процес запуску агента кодування

Використовуйте це на початку кожної сесії після завершення ініціалізації.

## Фіксований шаблон запуску

1. Виконайте `pwd` і підтвердьте кореневу директорію репозиторію.
2. Прочитайте `claude-progress.md`.
3. Прочитайте `feature_list.json`.
4. Перегляньте останні коміти за допомогою `git log --oneline -5`.
5. Виконайте `./init.sh`.
6. Запустіть базовий димовий тест або наскрізний сценарій.
7. Якщо базовий стан зламано, спочатку виправте це.
8. Виберіть незавершену функцію з найвищим пріоритетом.
9. Працюйте лише над цією функцією, доки вона не буде верифікована або явно заблокована.

## Чому важливий саме такий порядок

- `pwd` запобігає випадковій роботі в неправильній директорії.
- файли прогресу та функцій відновлюють довгостроковий стан до початку нових правок.
- останні коміти пояснюють, що змінювалось найостанніше.
- `init.sh` стандартизує запуск замість покладання на пам'ять.
- базова верифікація виявляє зламані початкові стани до того, як нова робота їх приховає.

## Дзеркало кінця сесії

Та сама сесія повинна завершуватись:

1. записом прогресу
2. оновленням стану функцій
3. написанням передачі за потреби
4. комітом безпечної роботи
5. залишенням чистого шляху для перезапуску
