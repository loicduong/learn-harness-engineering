# Калібрування підказок

Кореневі інструкції мають визначати операційну рамку, а не кожен можливий крок.

## Залишати в кореневому файлі

- призначення та обсяг репозиторію
- шлях запуску
- шлях верифікації
- незмінні обмеження
- обов'язкові артефакти стану
- правила завершення сесії

## Виносити з кореневого файлу

- тривалі історичні крайні випадки
- деталі реалізації для конкретних тем
- локальні архітектурні нотатки, що належать поруч із кодом
- приклади, що стосуються лише одного підсистемного компонента

## Робоче правило

Кореневий файл повинен допомагати новій сесії швидко орієнтуватись. Якщо файл
перетворюється на звалище для кожного минулого збою, розбийте деталі на менші
документи і посилайтесь на них.
