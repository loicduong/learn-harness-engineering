# Посібник агента-ініціалізатора

Використовуйте цей посібник для першої повноцінної сесії в репозиторії, до початку
поступової роботи над функціями.

## Мета

Створити стабільну робочу поверхню, щоб наступні сесії могли реалізовувати
поведінку без повторного виведення команд запуску, поточного статусу або меж задачі.

## Обов'язкові результати

Ініціалізатор повинен залишити після себе принаймні такі артефакти:

- кореневий файл інструкцій, наприклад `AGENTS.md` або `CLAUDE.md`
- машиночитаний перелік функцій, наприклад `feature_list.json`
- довгостроковий артефакт прогресу, наприклад `claude-progress.md`
- стандартний допоміжний сценарій запуску, наприклад `init.sh`
- початковий безпечний коміт, що фіксує базовий каркас

## Контрольний список

1. Визначте стандартний шлях запуску.
2. Визначте стандартний шлях верифікації.
3. Створіть журнал прогресу та запишіть початковий стан.
4. Декомпозуйте роботу на явні функції зі статусами.
5. Створіть перший чистий базовий коміт.

## Тест на успіх

Нова сесія без попереднього контексту чату повинна бути здатна відповісти на питання:

- що робить цей репозиторій
- як його запустити
- як верифікувати
- що не завершено
- який наступний найкращий крок
